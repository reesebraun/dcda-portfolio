Lab 2: AI Tool Evaluation

DCDA 40833 Skills Portfolio | Spring 2026

Overview

In this lab, I evaluated two AI tools—Canva AI and Claude—using a framework focused on capabilities, appropriate use, and ethical considerations...

Tool 1: Canva AI

I tested Canva AI’s design capabilities using accessibility-focused prompts to see how well it handles WCAG readability/contrast guidance, inclusive palettes, and more complex accessibility constraints. I documented each prompt, Canva’s output, and what the result suggests about the tool’s strengths and limits.

Evidence: Prompts + Outputs

Prompt 1: WCAG Accessibility (Baseline)

Prompt: “Design a social media post announcing a sustainability event that meets WCAG accessibility standards for color contrast and readability.”

Takeaway: Both outputs prioritize clean hierarchy and readable text, but accessibility is treated as a visual approximation rather than a verified standard.

Prompt 2: Color-blind Friendly + Low Vision + Alt Text

Prompt: “Can you use a color-blind friendly palette, readable hierarchy for low vision, and alt-text suggestions?”

Takeaway: Canva responds well to accessibility language, but may introduce new contrast issues—suggesting accessibility is treated as an aesthetic adjustment, not a measurable standard.

Prompt 3: WCAG AA Contrast Verification (Claims vs Proof)

Prompt: “Redesign this post so that all text meets WCAG AA contrast ratios. Please specify which text/background combinations meet the standard.”

Takeaway: Canva can redesign toward accessibility, but it does not audit or confirm WCAG AA compliance—verification still requires external checking and human review.

Prompt 4: Motion + Cognitive Accessibility (Limit Test)

Prompt: “Create a short animated version of this post that is accessible for users with cognitive disabilities and motion sensitivity.”

Takeaway: When accessibility constraints become multi-layered (motion sensitivity + cognition + text timing), Canva AI cannot fully execute the task and shifts responsibility back to the user.

Capabilities

Across the four tests, Canva AI was strongest at quickly producing polished layouts with clear hierarchy and a cohesive visual style. It responded to accessibility-oriented prompting by simplifying structure and increasing contrast in later iterations. However, the tool struggled with technical specificity: it did not provide measurable WCAG contrast proof, and it hit a clear limitation when asked to generate motion-accessible video that includes text. Overall, Canva AI behaves like a fast design assistant rather than an accessibility-aware system.

Appropriate Use

Canva AI is best used for early-stage drafting—quickly exploring layouts, visual directions, and accessibility-informed starting points. It becomes less appropriate when a project requires verified standards (WCAG AA compliance), nuanced accessibility needs (motion sensitivity), or accurate documentation of compliance. The most responsible workflow is: generate a draft with Canva AI, then refine intentionally and validate accessibility using external tools and human judgment.

Ethical Considerations

The biggest ethical risk I observed is “confidence without verification.” Canva AI can present outputs as accessible, but it does not reliably confirm compliance or provide evidence (contrast ratios, verified pairings). In public-facing or institutional communication, this can create a false sense of inclusion. Accessibility is not just a visual style—it affects who can participate. These tests reinforced that designers must take responsibility for validation and avoid treating AI-generated accessibility as automatically correct.
